# Таблица дефектов Notification Hub

| № | Дефект | Причина | Решение | Commit |
|---|--------|---------|---------|--------|
| **1** | Worker не обрабатывает задачи автоматически | Нет периодического вызова /process | Добавлен setInterval в фронтенд для вызова /process каждые 5 секунд | [fix: auto-process worker tasks] |
| **2** | CORS ошибка при запросах из браузера | Flask модули не поддерживают CORS | Добавлен flask-cors во все модули | [fix: add CORS support] |
| **3** | Ошибка 500 при сохранении в БД без запущенного PostgreSQL | Отсутствует проверка подключения к БД | Добавлена обработка исключений и fallback режим | [fix: database connection error handling] |
| **4** | При ошибке валидации уведомление не логируется | Логирование только после успешной валидации | Логирование ошибок перенесено в начало обработчика | [fix: log validation errors] |
| **5** | Worker падает при получении неизвестного канала | Отсутствует обработка default в switch | Добавлен default с записью в лог | [fix: unknown channel handling] |

## Детальное описание дефектов

### Дефект #1: Worker не обрабатывает задачи автоматически
- **Симптом:** Уведомление попадает в очередь, но не отправляется
- **Причина:** Worker требует ручного вызова /process
- **Решение:** Фронтенд вызывает /process каждые 5 секунд

### Дефект #2: CORS ошибка
- **Симптом:** Браузер блокирует запросы, статусы модулей красные
- **Причина:** Flask не разрешает跨域请求
- **Решение:** Установлен flask-cors и добавлен CORS(app)

### Дефект #3: Ошибка БД
- **Симптом:** Падение модуля при недоступной БД
- **Причина:** Нет try-except блока
- **Решение:** Добавлена обработка исключений

### Дефект #4: Потеря логов
- **Симптом:** Ошибочные запросы не видны в логах
- **Причина:** Логирование после валидации
- **Решение:** Логирование перенесено в начало

### Дефект #5: Неизвестный канал
- **Симптом:** Worker падает с ошибкой
- **Причина:** Нет обработки default
- **Решение:** Добавлен default с логированием
